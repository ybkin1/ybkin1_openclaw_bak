## 🏆 一、模型推荐

### 1.1 综合最佳：Qwen3 系列

**供应商**: 阿里巴巴 (通义千问)  
**为什么首选 Qwen**:

| 优势 | 说明 |
|------|------|
| ✅ **中国境内可用** | 阿里云部署，无网络限制 |
| ✅ **完全开源** | Apache 2.0 许可，可商用 |
| ✅ **中文优化** | 中文对话能力最强 |
| ✅ **多尺寸可选** | 0.5B / 1.5B / 3B / 7B / 14B / 32B / 72B |
| ✅ **易部署** | Ollama / vLLM / SGLang 一键部署 |

| 模型 | 显存需求 | 推理速度 | 对话质量 | 适合场景 |

**🎯 推荐：Qwen3-14B** - 16GB 显存可运行，对话质量接近 32B，成本只有 1/3

│                                                         │

│  │  文档     │    │  引擎     │    │   14B       │      │

**推荐工具栈**:
| 组件 | 工具 | 理由 |
|------|------|------|
| 模型部署 | Ollama | 一键部署，API 兼容 |
| RAG 引擎 | AnythingLLM | 开源，支持中文，Web UI |
| 向量库 | Milvus Lite | 轻量，单文件 |
| 嵌入模型 | Qwen3-Embedding | 中文优化 |
| 客服 UI | 自研 / Botpress | 可定制 |

**部署命令**:
```bash
# 1. 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2. 下载 Qwen3-14B
ollama pull qwen3:14b

# 3. 安装 AnythingLLM (Docker)
docker run -d -p 3001:3001 \
  -v /data/anythingllm:/app/server/storage \
  docker.anythingllm/anythingllm

# 4. 配置 RAG
# 访问 http://localhost:3001
# 添加知识库文档，选择 Qwen3 模型
```

- https://github.com/mrmendoza-dev/offline-chatbot

### 3.2 知识库准备

**文档类型**:
| 类型 | 格式 | 示例 |
|------|------|------|
| FAQ | Markdown/Excel | 常见问题 100 条 |
| 产品手册 | PDF/Word | 产品规格、使用说明 |
| 维修指南 | PDF | 故障排查步骤 |
| 历史工单 | CSV/Excel | 过往客服记录 |
| 政策文档 | Word | 退换货政策、保修条款 |

**处理流程**:
```
原始文档 → 文本提取 → 智能分块 → 向量化 → 存入向量库
```

**推荐工具**:
- **文档解析**: Unstructured / LangChain
- **分块**: LangChain TextSplitter (按语义分块)
- **嵌入**: Qwen3-Embedding
- **向量库**: Milvus / Chroma

## 🔧 四、微调方案

### 4.1 什么时候需要微调？

| 场景 | 是否需要微调 | 说明 |
|------|-------------|------|
| 通用客服 | ❌ 不需要 | RAG 足够 |
| 专业术语多 | ✅ 需要 | 如服务器硬件术语 |
| 固定回复格式 | ✅ 需要 | 如工单格式 |
| 多轮对话优化 | ✅ 需要 | 提升对话连贯性 |
| 品牌语气 | ✅ 需要 | 统一回复风格 |

---

### 4.2 推荐微调方案：LoRA

**为什么选 LoRA**:
- ✅ **显存需求低** - 14B 模型只需 24GB 显存
- ✅ **训练快** - 几小时完成
- ✅ **效果好** - 接近全量微调
- ✅ **可切换** - 同一基座可加载多个 LoRA

**训练数据准备**:
```json
[
  {
    "instruction": "用户询问 GPU 过热怎么办",
    "input": "服务器 GPU 温度达到 85 度，是否正常？",
    "output": "GPU 正常工作温度为 30-80 度。85 度略高，建议：\n1. 检查机房空调温度设置\n2. 清理 GPU 风扇灰尘\n3. 使用 nvidia-smi 查看 GPU 负载\n4. 如持续高温，请联系硬件运维\n\n当前温度已记录，如 1 小时内未下降将自动创建工单。"
  }
]
```

**训练命令** (使用 LLaMA-Factory):
```bash
# 安装
git clone https://github.com/hiyouga/LLaMA-Factory
cd LLaMA-Factory

# 准备数据 (格式转换为 alpaca)
# data/customer_service.json

# 启动训练 (LoRA)
llamafactory-cli train \
  --model_name_or_path Qwen/Qwen3-14B \
  --dataset customer_service \
  --finetuning_type lora \
  --lora_target q_proj,v_proj \
  --lora_rank 64 \
  --per_device_train_batch_size 4 \
  --gradient_accumulation_steps 4 \
  --lr_scheduler_type cosine \
  --logging_steps 10 \
  --save_steps 100 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --output_dir ./qwen3-cs-lora \
  --plot_loss
```

**训练成本估算**:

| GPU 租赁 (4xA100, 8 小时) | ¥400 |
| 数据存储 | ¥50 |
| 人力 (标注 1000 条) | ¥500 |
