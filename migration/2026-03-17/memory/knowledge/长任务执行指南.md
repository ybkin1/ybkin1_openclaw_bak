# 长任务执行指南

## 长任务识别

以下情况属于长任务：
- ⏱️ 预计执行时间 > 2 分钟
- 📦 需要安装多个依赖
- 🔄 涉及多次网络请求（如 clawhub 限流）
- 💾 大量文件操作（备份、扫描）
- 🐳 Docker 容器部署

## 执行策略

### 1. 后台执行（推荐）
```bash
# 使用 yieldMs 后台运行
exec command="..." yieldMs=10000

# 或用 sleep + 后台任务
sleep 5 && long_command &
```

### 2. 分步执行
```bash
# 步骤 1：准备
prepare_step

# 步骤 2：执行（后台）
execute_step --background

# 步骤 3：轮询检查
process action=poll sessionId=xxx timeout=60000
```

### 3. 限流处理
```bash
# clawhub 限流时等待 30-90 秒
sleep 30 && clawhub install <skill>

# 批量安装增加间隔
clawhub install skill1 && sleep 30 && clawhub install skill2
```

## Token 优化

### 减少浪费的方法

| 方法 | 说明 | 节省 |
|------|------|------|
| **后台执行** | 长任务不阻塞会话 | ~50% |
| **精简输出** | 成功时输出 `NO_REPLY` | ~30% |
| **分离子任务** | 用 sub-agent 处理独立任务 | ~40% |
| **避免重复** | 用记忆系统避免重复解释 | ~60% |

### 会话上下文管理

```
当前会话上下文限制：200k tokens
实际使用应控制在：35k-50k tokens

策略：
1. 重要决策 → 写入 memory/daily/
2. 学到的知识 → 写入 memory/knowledge/
3. 错误和解决 → 写入 .learnings/ERRORS.md
4. 会话结束 → 清理临时上下文
```

## Sub-agent 使用

### 何时使用
- 独立的研究任务
- 代码审查/扫描
- 批量处理任务
- 后台监控

### 示例
```bash
# 后台扫描 skills
sessions_spawn runtime="subagent" \
  task="用 skill-vetting 扫描所有 skills" \
  mode="run"
```

## 最佳实践

1. ✅ 长任务立即后台执行
2. ✅ 每 30 秒报告进度（避免超时）
3. ✅ 失败时记录到 .learnings/ERRORS.md
4. ✅ 完成后写入 memory/daily/
5. ❌ 避免在单会话中执行 >10 分钟的任务
6. ❌ 避免重复执行已成功的任务
