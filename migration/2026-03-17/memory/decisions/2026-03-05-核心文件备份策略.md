## 🤔 背景

用户需求：
1. 每天自动将 SOUL.md、MEMORY.md、AGENTS.md、USER.md 推送到 GitHub
2. 修改 MEMORY.md 前必须先备份为 MEMORY.md.bak

风险考虑：
- MEMORY.md 包含重要记忆，误操作可能导致数据丢失
- 需要可靠的备份机制确保可恢复
- GitHub 推送需要处理认证和网络问题

---

## 💡 实施方案

### 1. MEMORY.md 备份机制

**脚本**: `scripts/edit-memory.sh`

```bash
# 编辑前自动备份
./scripts/edit-memory.sh edit "[修改内容]"

# 或手动备份
./scripts/edit-memory.sh backup
```

**备份位置**:
- 最新备份：`MEMORY.md.bak` (工作区根目录)
- 历史备份：`backups/memory/MEMORY.md.YYYYMMDD_HHMMSS.bak`

**保留策略**: 7 天

### 2. 每日自动备份

**脚本**: `scripts/daily-backup.sh`

**Cron**: 每天凌晨 2:00

**备份内容**:
- SOUL.md
- MEMORY.md
- AGENTS.md
- USER.md

**打包**: `backups/daily/core-files.YYYY-MM-DD.tar.gz`

**保留策略**: 30 天

### 3. GitHub 自动推送

**脚本**: `scripts/push-core-files.sh`

**Cron**: 每天凌晨 3:00

**推送文件**:
- SOUL.md
- MEMORY.md
- AGENTS.md
- USER.md

**配置**: 需要用户提供 GitHub 仓库 URL

### 4. 每周完整备份

**Cron**: 每周日凌晨 4:00

**备份内容**: 整个 `memory/` 目录

**打包**: `backups/weekly/memory-YYYYMMDD.tar.gz`

## 📊 Cron 任务

```cron
# 每日备份
0 2 * * * daily-backup.sh

# GitHub 推送
0 3 * * * push-core-files.sh

# 每周完整备份
0 4 * * 0 tar memory/

# 清理过期备份
0 5 * * * find -mtime +30 -delete
```

## 🔒 安全提醒

1. **使用私有仓库** - 核心文件可能包含敏感信息
2. **定期轮换 Token** - 建议每 90 天更新
3. **监控推送日志** - 定期检查 `logs/github-push.log`
4. **保留本地备份** - 不完全依赖 GitHub
